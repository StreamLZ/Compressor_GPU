## ToDo.md: items deferred during Cluster A-D execution

* **DevBuf abstraction** (was "structural - DecodeContext + EncodeContext
  bloat"). High churn, low marginal win once `ensureDeviceBuf` returns
  `GpuError!void`. Every kernel-arg slot reads the raw `.ptr` (the
  CUDA kernel-params array needs a `&local: u64`), so encapsulating
  the (ptr, size) pair behind a struct gains us a clean `.ensure`
  call at the alloc site and loses us a `.ptr` everywhere else.
  Decision deferred to a future pass with a different cost/benefit
  picture.

* **slzCompressAsync uncompressed-body D2H** (was tagged in
  `streamlz_gpu.zig` comment when wrapping the sentinel). When the
  caller's input falls outside `[128, ~1 GiB]`, the C ABI currently
  returns `SLZ_ERROR_UNSUPPORTED` so they retry on the host-bounce
  entry point. A future revision could D2H from `d_input` and run
  the uncompressed-body path in-line, but that requires plumbing
  `d_input_override` into `fast_framed.writeUncompressedFrame` and
  is not justified by the threshold's narrow width.

* **Legacy CPU-codec wire-format types 1/5/6/7** in
  `decode/lz_header_parse.cuh` and `parallel_decode_metadata` block
  skipping in `streamlz_decoder.zig` — kept for backward compat with
  old `.slz` files produced by the deleted CPU encoder. ~150 lines
  total; a judgment call rather than dead code.

* **C ABI defaults mismatch documented** but not aligned
  (`Options.level = 1` on Zig encoder vs `slzCompressOpts_t.level =
  5` in the C header). Documented as intentional in
  `encode/streamlz_encoder.zig` Options docstring.
